# Testing Checklist

## Фаза 4: Manual Testing

### 4.1 Basic Functionality

#### Worker Initialization
- [ ] Откройте `index.html` в браузере (рекомендуется Chrome/Edge)
- [ ] Откройте Developer Console (F12)
- [ ] Проверьте, что нет ошибок в консоли
- [ ] Убедитесь, что Worker загружен (сообщение "PLC Worker Started")
- [ ] Проверьте, что HMI инициализирован (сообщение "HMI Initialized")

#### Data Blocks
- [ ] В консоли проверьте InputDB: `console.log(InputDB)` (в worker console)
- [ ] Проверьте OutputDB: `console.log(OutputDB)` (в worker console)
- [ ] Проверьте HmiDB: `console.log(HmiDB)` (в worker console)
- [ ] Убедитесь, что все параметры инициализированы с правильными значениями по умолчанию

#### Communication
- [ ] Проверьте, что пост-сообщения отправляются (в network tab)
- [ ] Убедитесь, что PLC_STATE_UPDATE приходит каждые 16.7ms (60 FPS)
- [ ] Проверьте payload в сообщениях (outputs и hmi)

### 4.2 Manual Mode

#### Valve Control
- [ ] Переключитесь в Manual режим (кнопка Manual)
- [ ] Двигайте слайдер клапана от 0% до 100%
- [ ] Проверьте, что OutputDB.actuator_SteamValvePercent обновляется
- [ ] Проверьте, что процесс реагирует на изменения (температуры меняются)
- [ ] Проверьте bumpless transfer: Manual → Auto → Manual (не должно быть скачков)

#### Temperature Response
- [ ] Установите клапан на 50%
- [ ] Дождитесь стабилизации температур
- [ ] Установите клапан на 80%
- [ ] Дождитесь повышения температур
- [ ] Установите клапан на 20%
- [ ] Дождитесь понижения температур
- [ ] Проверьте, что IAE накапливается

### 4.3 Auto Mode - Single Loop

#### Basic Operation
- [ ] Переключитесь в Single режим (кнопка Single)
- [ ] Переключитесь в Auto режим (кнопка Auto)
- [ ] Убедитесь, что SP = 72°C (по умолчанию)
- [ ] Проверьте, что PID управляет клапаном
- [ ] Проверьте, что температура сходится к 72°C

#### PID Tuning
- [ ] Kp = 1.0: медленная реакция
- [ ] Kp = 5.0: быстрая реакция, возможны перерегулирования
- [ ] Kp = 2.0, Ti = 4.0: сбалансированная реакция
- [ ] Kp = 2.0, Ti = 2.0, Td = 0.5: быстрая реакция с демпфированием

#### Setpoint Changes
- [ ] Измените SP с 72°C на 65°C
- [ ] Проверьте, что температура плавно падает
- [ ] Измените SP с 65°C на 78°C
- [ ] Проверьте, что температура плавно растет
- [ ] Верните SP = 72°C

### 4.4 Auto Mode - Cascade

#### Basic Operation
- [ ] Переключитесь в Cascade режим (кнопка Cascade)
- [ ] Проверьте, что Master PID (T_milk) управляет SP для Slave
- [ ] Проверьте, что Slave PID (steam pressure) управляет клапаном
- [ ] Убедитесь, что температура сходится к 72°C

#### Response vs Single
- [ ] Сравните время отклика на изменение SP:
  - Cascade: быстрее, меньше перерегулирование
  - Single: медленнее, больше перерегулирование
- [ ] Сравните устойчивость к возмущениям:
  - Cascade: более устойчивая
  - Single: менее устойчивая

### 4.5 Disturbances

#### Steam Pressure Drop
- [ ] Нажмите кнопку "Steam Pressure Drop"
- [ ] Проверьте, что температура падает
- [ ] Проверьте, что PID компенсирует возмущение (в Auto режиме)
- [ ] Нажмите снова, чтобы отключить возмущение

#### Valve Wear (Deadband)
- [ ] Установите Valve Wear на 3%
- [ ] Убедитесь, что клапан игнорирует малые изменения задания (менее 3%)
- [ ] Проверьте возникновение автоколебаний (limit cycles) при большом износе
- [ ] Верните Valve Wear на 0%

#### Pressure Noise (±BAR)
- [ ] Установите Pressure Noise на 0.5 BAR
- [ ] Проверьте, что значение давления пара на HMI начало "гулять" (псевдослучайно)
- [ ] Убедитесь, что ПИД-регулятор реагирует на эти изменения
- [ ] Проверьте, что шум фильтруется и не вызывает мгновенных скачков клапана

#### PID Deadband
- [ ] Установите Deadband (°C) = 0.2 для активного регулятора
- [ ] Дождитесь входа PV в зону [SP-0.2, SP+0.2]
- [ ] Убедитесь, что выход клапана (Y101) замер (линия на графике стала идеально прямой)
- [ ] Измените SP так, чтобы выйти из зоны нечувствительности, и проверьте восстановление управления

### 4.6 Speed Multiplier

#### Speed Test
- [ ] Проверьте список скоростей: Pause, Normal, Quick, Fast, Turbo, Ludicrous (20x)
- [ ] Установите Speed = Pause (0x): симуляция должна полностью замереть
- [ ] Установите Speed = Ludicrous (20x): проверьте отсутствие лагов UI при экстремальной скорости
- [ ] Верните Speed = Normal (1x)

#### Reset Logic
- [ ] Нажмите "Reset Scene"
- [ ] Убедитесь, что время сбросилось в 0, а температуры — к 20°C
- [ ] **ВАЖНО**: Проверьте, что ваши коэффициенты ПИД (Kp, Ti, Td) сохранились!
- [ ] Убедитесь, что режим переключился в Manual
- [ ] Проверьте кнопку Reset в блоке IAE — она должна обнулять только счетчик ошибки, не трогая симуляцию

#### IAE "Touch-Startup" Verification
- [ ] Нажмите "Reset Scene"
- [ ] Включите режим "1 PID"
- [ ] Наблюдайте за счетчиком IAE (Error Sum)
- [ ] Убедитесь, что он равен 0.00, ПОКА температура не коснулась уставки (Setpoint)
- [ ] Как только PV >= SP, счетчик должен начать активный отсчет
- [ ] Проверьте точность: при PV=SP ошибка не должна расти

### 4.7 Charts

#### uPlot Rendering
- [ ] Убедитесь, что график отображается
- [ ] Проверьте, что T_milk линия синяя (#3b82f6)
- [ ] Проверьте, что Setpoint линия красная пунктирная (#ef4444)
- [ ] Проверьте, что T_water линия зеленая (#10b981)
- [ ] Проверьте, что Valve % линия оранжевая (#f59e0b)
- [ ] Проверьте rolling window: 60 секунд

#### Real-time Updates
- [ ] Проверьте, что данные обновляются плавно
- [ ] Убедитесь, что нет пропусков данных
- [ ] Проверьте масштаб по оси Y (0-100°C)
- [ ] Проверьте масштаб по оси X (время)

### 4.8 Performance

#### Frame Rate
- [ ] Проверьте 60 FPS (используйте Performance Monitor)
- [ ] Проверьте CPU usage (должен быть < 30%)
- [ ] Проверьте Memory (не должно быть утечек)
- [ ] Проверьте Network (постоянный поток сообщений)

#### Optimization
- [ ] Проверьте, что DOM обновления не блокируют UI
- [ ] Проверьте, что Worker не блокирует UI
- [ ] Проверьте, что график рендерится быстро (< 10ms)

### 4.9 SVG HMI

#### Dynamic Elements
- [ ] Проверьте, что температура молока отображается (T_milk)
- [ ] Проверьте, что температура воды отображается (T_water)
- [ ] Проверьте, что клапан анимируется при изменении
- [ ] Проверьте цветовую индикацию:
  - Зеленый: error < 1°C
  - Желтый: error < 3°C
  - Красный: error >= 3°C
